# Running the Code
**Only Bash Terminal Commands are Provided. Using Deepdish Server is Recommended. The Given Commands May Need to be Adjusted if not Run on Deepdish Server** 

## Prerequisites
1. Clone this repo
2. Navigate to the repo root
3. Create and activate a new environment
```bash
python3 -m venv .venv
```
```bash
source .venv/bin/activate
```
4. Create the .env File
```bash
cp .env.example .env
``` 
**Fill in Your Pinecone and OpenAI API Keys, do not touch other fields (DO NOT PUT YOUR API KEYS IN QUOTES)**
  
## Install PyTorch (CPU or GPU)

This project requires PyTorch. Install **one** of the following:

### NVIDIA GPU (CUDA 12.1) (RECOMMENDED)
```bash
pip install torch --index-url https://download.pytorch.org/whl/cu121
```
**If you need a different CUDA version, use the official PyTorch install selector and choose the CUDA version that matches your system.**  

### CPU (works everywhere)
```bash
pip install torch --index-url https://download.pytorch.org/whl/cpu
```

## Install Packages
```bash
pip install -r requirements.txt
```

## Run the Code
```bash
python -m src.main --gpu 1 --verbose
```
**YOU MUST SPECIFY A DEVICE. THE COMMAND GIVEN DEFAULTS TO GPU 1. IF NO DEVICE IS SPECIFIED, THE CODE WILL NOT RUN EFFICIENTLY OR CORRECTLY**  
  
You can optionally run without the verbose flag to see less progress in the terminal.
  
##### What the Pipeline Does

The pipeline executes the following steps:
1. Scrapes Steam reviews for the selected games
2. Chunks the reviews and generates embeddings for each chunk
3. Initializes the Pinecone vector database and index
4. Uploads the embedded review chunks to Pinecone
5. Builds a training and evaluation dataset for LoRA fine-tuning
6. Fine-tunes a LoRA adapter on the review-based dataset
7. Runs a set of evaluation questions across four systems:
   - Base LLM (no retrieval)
   - Basic RAG
   - Advanced agentic RAG with a base model
   - Advanced agentic RAG with the LoRA-fine-tuned model
8. Uses an LLM judge to automatically evaluate the responses and compare system performance
9. Writes evaluation results and rankings to the `outputs/evaluations/` directory
10. Launches an interactive mode for querying the agentic RAG system


# Outputs Produced
After running the code, the following output directories and files are generated:
```bash
outputs/
  responses/
    assignment_outputs_<timestamp>.txt
  evaluations/
    evaluation_<timestamp>.json
    evaluation_<timestamp>.txt
  steam_lora_adapter/
```

**outputs/responses/assignment_outputs_<timestamp>.txt**  
Contains the responses to the evaluation questions generated by the four systems:  
  
1. Base LLM (no RAG)
2. Basic RAG
3. Advanced agentic RAG with the base model
4. Advanced agentic RAG with the LoRA fine-tuned model.  
  
**outputs/evaluations/evaluation_<timestamp>.json**  
Structured evaluation results produced by the LLM judge, including per-question scores (groundedness, completeness, nuance, clarity, and overall quality) for each system.  
  
**outputs/evaluations/evaluation_<timestamp>.txt**  
A human-readable evaluation report summarizing the scores, system rankings, and qualitative comparisons across the evaluated questions.  
  
**outputs/steam_lora_adapter/**  
Directory containing the trained LoRA adapter weights produced during fine-tuning.
